## Do less. Earn more.

Superform is smart account earn infrastructure for the onchain economy. Deposit into optimized vaults, execute multi-chain strategies with a single signature, and let curators compete to deliver the best risk-adjusted yields.

The protocol is a suite of persistent, non-upgradeable smart contracts that compress arbitrary on-chain actions into a single, Merkle-verified signature. Users interact through ERC-7579 Smart Accounts, and composition comes from plug-and-play Hooks that anyone can deploy.

## How It Works

1. **User signs one transaction** — the frontend builds a bundle of hooks (swap USDC → deposit into Morpho → bridge to Base) and presents it as a single signature request.
2. **SuperBundler packages the bundle** — splits hooks per chain, wraps each into an ERC-4337 UserOperation, and stores a Merkle tree of all authorized operations.
3. **Smart Account validates** — the on-chain SuperValidator checks the Merkle proof. If valid, the SuperExecutor iterates through hooks atomically.
4. **Accounting updates** — SuperLedger records cost basis and position changes for trustless performance fee calculation.

If any hook in a chain fails, the entire transaction reverts. Atomicity is guaranteed.

## Core Components

### Execution Layer

ERC-7579 executors (SuperExecutor, SuperDestinationExecutor) process hook bundles with transient storage for gas-efficient inter-hook communication. Same-chain bundles are atomic; cross-chain bundles use bridge adapters with destination validators.

### Validation Layer

Merkle-proof validators (SuperValidator, SuperDestinationValidator) enable single-signature authorization for batched multi-chain operations. One Merkle root encodes every permitted action, so no per-transaction approval is needed.

### Accounting Layer

SuperLedger and YieldSourceOracles provide trustless cost basis tracking and performance fee calculation across vault standards (ERC-4626, ERC-5115, ERC-7540, Pendle).

### Infrastructure

- **Bridge adapters** for cross-chain messaging (Across V3, deBridge, Circle)
- **SuperNativePaymaster** for ERC-20 gas sponsorship (pay gas in USDC)
- **SuperBundler** for batched UserOperation processing

## Architecture at a Glance

Superform compresses arbitrary multi-chain actions into a single Merkle-verified signature. The stack:

- **Smart Accounts** — ERC-7579 accounts with gas abstraction, passkey login, and session keys
- **Hooks** — Permissionless on-chain modules (bridge, swap, lend) that chain together into strategies
- **SuperBundler** — Batches hook chains into UserOperations with Merkle proofs
- **SuperVaults** — ERC-7540 tokenized vaults where curators manage yield allocation
- **$UP** — Protocol token for upkeep, validator bonding, staking, and governance
