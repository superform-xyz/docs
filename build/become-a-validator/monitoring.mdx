## Health Endpoints

Both ports are configured under `[monitoring]` in `config.toml`.

| Endpoint | Port | Response |
|----------|------|----------|
| `GET /metrics` | `metrics_port` (9090) | Prometheus metrics in OpenMetrics format |
| `GET /healthz` | `health_port` (8080) | `200 OK` when running and DB ping succeeds; `503` during shutdown |
| `GET /readyz` | `health_port` (8080) | Same checks as `/healthz` today; reserved for future consensus readiness gates |

Quick sanity check after startup:

```bash
curl -s localhost:8080/healthz
curl -s localhost:8080/readyz
curl -s localhost:9090/metrics | grep ^ocr2_ | head -20
```

## Prometheus Metrics

All series are prefixed `ocr2_`. Labels are low-cardinality: `chain_id`, `strategy`, `phase`, `result`, `kind`, `op`, `status`, `stage`.

### On-Chain Transmission Health

These are the most important metrics for verifying that PPS updates are actually landing on-chain.

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_strategy_last_update_seconds{chain_id, strategy}` | Gauge | Unix timestamp of the last observed `PPSValidated` event. Stops advancing if the tx reverts on-chain. |
| `ocr2_strategy_update_stale{chain_id, strategy}` | Gauge | `1` if no `PPSValidated` event within `health_check_interval`, `0` otherwise. |

**Alert on** `ocr2_strategy_update_stale == 1` for more than one `health_check_interval`.

### Transmitter

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_transmit_total{chain_id, status}` | Counter | Tx submissions. `status` ∈ `success`, `failure`, `error` |
| `ocr2_transmit_strategies_total{chain_id}` | Counter | Strategies packed into each tx |
| `ocr2_transmit_pack_errors_total{chain_id}` | Counter | Calldata pack failures (missing domain separator or no valid signatures) |
| `ocr2_transmit_duration_seconds{result}` | Histogram | End-to-end transmit latency |

### Plugin Phases

`phase` ∈ `query`, `observation`, `report`, `should_accept`, `should_transmit`

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_plugin_phase_duration_seconds{phase, result}` | Histogram | Per-phase latency |
| `ocr2_plugin_phase_total{phase, result}` | Counter | Per-phase invocation count. `result` ∈ `success`, `failure`, `error`, `accept`, `transmit`, `stale`, `deviation`, `nonce_mismatch`, `invalid_pps` |
| `ocr2_plugin_observations_total{chain_id, result}` | Counter | Per-strategy PPS observation outcomes (`success`, `error`, `skipped`) |
| `ocr2_plugin_insufficient_observations_total{chain_id}` | Counter | Strategies dropped from the report because fewer than quorum validators observed them |
| `ocr2_plugin_stale_reports_total{phase}` | Counter | Reports rejected for exceeding `max_stale_duration` |
| `ocr2_plugin_deviation_rejections_total` | Counter | Reports rejected because the leader's PPS exceeded `max_pps_deviation_ppb` from this node's observation |
| `ocr2_plugin_nonce_mismatch_total` | Counter | Reports rejected because the on-chain nonce advanced between report creation and acceptance |
| `ocr2_plugin_pps_deviation_ppb{chain_id, strategy}` | Gauge | Most recent deviation between the leader's PPS and this node's observation, in parts per billion |

### Validator Configuration Tracker

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_config_changes_total` | Counter | `ValidatorConfigSet` events applied |
| `ocr2_config_reorgs_total` | Counter | Events rejected due to non-increasing version (reorg replay) |
| `ocr2_config_load_errors_total{stage}` | Counter | Config load failures. `stage` ∈ `parse`, `reload` |
| `ocr2_config_version` | Gauge | Current applied config version |
| `ocr2_config_f` | Gauge | Current F (max faulty validators) |
| `ocr2_config_signers` | Gauge | Current number of registered signers |

After a governance update, verify all nodes report the same `ocr2_config_version` and `ocr2_config_signers`.

### Keyring

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_sign_total{kind, result}` | Counter | Signing operations. `kind` ∈ `onchain`, `offchain` |
| `ocr2_sign_duration_seconds{kind}` | Histogram | Signing latency. Watch `onchain` if using AWS KMS — KMS latency spikes surface here. |

### Pricing Service

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_pps_fetch_total{chain_id, result}` | Counter | PPS fetch outcomes. `result` ∈ `success`, `error`, `zero` |
| `ocr2_pps_fetch_duration_seconds{chain_id}` | Histogram | RPC latency for the PPS call |

### Database

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_db_query_duration_seconds{op, result}` | Histogram | Per-operation DB latency |
| `go_sql_*` | Various | Connection pool stats (`in_use`, `idle`, `wait_count`, `wait_duration`). Sustained `wait_count` growth is the early signal for pool exhaustion. |

### P2P Network

libOCR's ragep2p metrics appear with `libocr_`, `ragep2p_`, and `rageping_` prefixes. They cover inbound/outbound messages, dial attempts, reconnects, and per-peer ping RTT histograms.

| ragep2p Default | Value | Effect |
|-----------------|-------|--------|
| `DeltaReconcile` | 10s | Peer list reconciliation interval across the mesh |
| `DeltaDial` | 1s | Minimum spacing between dial attempts to the same peer |
| `IncomingMessageBufferSize` | 100 | Dropped inbound messages surface as warn logs |
| `OutgoingMessageBufferSize` | 100 | Dropped outbound messages surface the same way |

### Liveness

| Metric | Type | Description |
|--------|------|-------------|
| `ocr2_up` | Gauge | `1` while serving, `0` during shutdown |
| `ocr2_build_info{version}` | Gauge | Build identifier, constant `1` |

## Recommended Alerts

| Alert | Query | Severity |
|-------|-------|----------|
| PPS update stale | `ocr2_strategy_update_stale == 1` for > 1 interval | Critical |
| Observation errors rising | `rate(ocr2_plugin_observations_total{result="error"}[15m]) > 0` | Warning |
| Transmit failures | `rate(ocr2_transmit_total{status=~"failure|error"}[15m]) > 0` | Warning |
| Node down | `ocr2_up == 0` | Critical |
| KMS signing latency | `histogram_quantile(0.99, ocr2_sign_duration_seconds{kind="onchain"}) > 2` | Warning |
| DB pool exhaustion | `go_sql_wait_count > 5` sustained over 5m | Warning |
