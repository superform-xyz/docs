## Overview

The Supervaults Validator Network is a decentralized oracle network that secures price-per-share (PPS) data for SuperVaults. Validators run oracle nodes that independently observe vault PPS values, reach consensus using the Chainlink OCR2 protocol, and submit signed attestations on-chain.

The `ECDSAPPSOracle` contract accepts a PPS update only after at least two-thirds of weighted validator stake has signed the same value. No single validator can manipulate the reported price.

## What Validators Do

- **Observe** — Fetch PPS values from vaults on each configured chain
- **Attest** — Sign EIP-712 reports using a dedicated ECDSA key
- **Transmit** — Submit aggregated, multi-validator reports to the oracle contract
- **Earn** — Receive a share of $UP upkeep payments from vault curators

## Network Architecture

The network relies on four cryptographic keys per node, two infrastructure bootstrap peers, and a `SuperGovernor` contract that is the authoritative source for the active validator set.

| Key | Algorithm | Purpose |
|-----|-----------|---------|
| Offchain | Ed25519 | Signs OCR2 protocol messages between validators |
| P2P | Ed25519 | Network identity for peer discovery |
| Config | X25519 | Encrypted config distribution (derived from offchain key) |
| Onchain | ECDSA secp256k1 | Signs PPS updates verified on-chain |

Changes to the validator set take effect the moment a new `ValidatorConfigSet` event is emitted from `SuperGovernor` — no node restarts required.

## Requirements

- **$UP stake** — Validators must bond $UP tokens to join the validator set
- **Infrastructure** — A server running the oracle node binary with PostgreSQL and stable network access
- **Uptime** — Consistent participation is required; insufficient participation triggers slashing

## Economics

- Validators earn a share of $UP upkeep payments from vault curators
- Misreporting is deterred through slashing of bonded $UP
- An insurance fund backstops depositors against oracle failures
