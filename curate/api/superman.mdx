<Warning>
The Superman Python backend **does not have an OpenAPI spec**. This reference is manually maintained from source code analysis. Endpoint shapes are accurate as of March 2026 but may drift. OpenAPI spec generation is a priority for the Superform team.
</Warning>

Superman is a Python (Celery + FastAPI) service powering the operational layer of SuperVaults: ladder configuration, vault state, execution history, keeper service health, and swap slippage configuration.

**Base URL:** `https://superman.superform.xyz`
**Auth:** `Authorization: Bearer <jwt>` (same JWT as Erebor)
**OpenAPI spec:** Not yet available

---

## Ladder Endpoints

All under prefix `/ladder`.

### Get All Ladders

```
GET /ladder/ladders
```

Returns all ladder configurations. Filter by `vault_address` and `chain_id` client-side.

```json
[
  {
    "vault_address": "0x...",
    "chain_id": 8453,
    "deposit_priority": [...],
    "redemption_priority": [...],
    "reserve_percent": 5,
    "is_enabled": true
  }
]
```

### Update Deposit Priority

```
PUT /ladder/{chain_id}/{vault_address}/deposit-priority
{ "deposit_priority": [{ "address": "0xYieldSource1..." }, { "address": "0xYieldSource2..." }] }
```

### Update Redemption Priority

```
PUT /ladder/{chain_id}/{vault_address}/redemption-priority
{ "redemption_priority": [{ "address": "0xYieldSource1...", "min_balance_to_keep": "1000000000000000000" }] }
```

`min_balance_to_keep` is in wei.

### Update Reserve

```
PUT /ladder/{chain_id}/{vault_address}/reserve
{ "reserve_percent": 5 }
```

Range: 0-100.

### Claim Rewards Ladder

```
GET /ladder/{chain_id}/{vault_address}/claim-rewards-ladder
PUT /ladder/{chain_id}/{vault_address}/claim-rewards-ladder
{
  "claim_rewards_ladder": [
    { "token_address": "0x...", "floor_amount": "1000000000000000000", "is_enabled": true }
  ]
}
```

### Claimable Rewards

```
GET /ladder/{chain_id}/{vault_address}/claimable-rewards
```

Returns current accumulated reward token balances.

### Swap Ladder

```
GET /ladder/{chain_id}/{vault_address}/swap-ladder
PUT /ladder/{chain_id}/{vault_address}/swap-ladder
{
  "swap_ladder": [
    { "token_address": "0x...", "floor_amount": "500000000", "is_enabled": true }
  ]
}
```

### Swappable Tokens

```
GET /ladder/{chain_id}/{vault_address}/swappable-tokens
```

### Validate Yield Source

```
GET /ladder/{chain_id}/{vault_address}/validate/{yield_source_address}
```

Validates whether a yield source address is valid for the given vault.

## Vault Endpoints

All under prefix `/vaults`.

### Get Vault List (Simple)

```
GET /vaults/simple
```

Returns a simplified list of all vaults.

### Get Vault List (Advanced)

```
GET /vaults/advanced
```

Returns detailed vault data including allocation breakdowns.

### Get Vault State

```
GET /vaults/{vault_address}
```

Returns full vault state including on-chain data, PPS, allocations, and manager info.

### Get PPS History

```
GET /vaults/{vault_address}/pps-history
```

Returns price-per-share history for the vault.

### Toggle Vault Enabled

```
PUT /vaults/{chain_id}/{vault_address}/enabled
{ "enabled": true }
```

Master switch for all keeper workers on this vault.

### Update Redeem Timelock

```
PUT /vaults/{chain_id}/{vault_address}/redeem-timelock
{ "redeem_timelock_seconds": 86400 }
```

Sets the redeem timelock for a vault. Set to `null` to remove the timelock (instant redemptions).

### Redemption Execution Detail

```
GET /fulfill-redeems/executions/{execution_id}
```

### Pending Redemptions

```
GET /fulfill-redeems/pending
```

Returns pending redemption requests awaiting fulfillment.

### Execute Fulfillment

```
POST /fulfill-redeems/execute
```

Triggers fulfillment for a specific vault's pending redemptions.

### Execute All Fulfillments

```
POST /fulfill-redeems/execute-all
```

Triggers fulfillment across all vaults with pending redemptions.

### Fulfillment Health

```
GET /fulfill-redeems/health
```

Returns health status of the redemption fulfillment system.

## Service Health Endpoints

All under prefix `/services`.

### Infrastructure Health

```
GET /health
```

```json
{
  "components": [
    { "component": "api", "status": "operational", "last_checked": "...", "message": null },
    { "component": "database", "status": "operational" },
    { "component": "redis", "status": "operational" },
    { "component": "celery", "status": "operational" },
    { "component": "subgraph", "status": "operational" },
    { "component": "rpc", "status": "operational" }
  ]
}
```

### All Service Statuses

```
GET /services/status
```

```json
{
  "services": [
    {
      "name": "fulfill_deposits",
      "display_name": "Fulfill Deposits",
      "status": "healthy",
      "last_run": "2025-03-01T10:00:00Z",
      "next_run": "2025-03-01T10:01:00Z",
      "schedule_seconds": 60,
      "success_rate": 0.98,
      "avg_execution_time": 4.2,
      "total_executions": 8400,
      "is_enabled": true
    }
  ]
}
```

### Single Service Status

```
GET /services/{service_name}/status
```

### Global Pause/Unpause

```
POST /services/pause
{ "service_name": "fulfill_deposits", "paused": true }
```

### Pause / Resume a Service

```
POST /services/{service_name}/pause
POST /services/{service_name}/resume
```

Valid names: `fulfill_deposits`, `fulfill_redeems`, `fulfill_claims`, `fulfill_swaps`, `redemption_queue_engine`, `cleanup_processed_redeems`, `tx_manager`

### Schedule Management

```
GET /services/schedules
GET /services/schedules/{task_name}
PUT /services/schedules/{task_name}
{ "interval_seconds": 60 }

POST /services/schedules/{task_name}/enable
POST /services/schedules/{task_name}/disable
```

<Note>
Schedule endpoints use `task_name` (the Celery task name), not `service_name`. Example: `fulfill_deposits`.
</Note>

### Scheduler Status

```
GET /services/scheduler/status
```

  "loaded_schedules": ["fulfill_deposits", "..."],

## Swap Slippage Configuration

### Update Swap Slippage

```
PUT /swap-slippage/{chain_id}/{vault_address}/slippage
{ "swap_slippage": 0.01 }
```

Sets the swap slippage tolerance for the `fulfill_swaps` worker (decimal: 0.01 = 1%). Must be `> 0` and `<= 1.0`.

### Verify JWT

## Error Codes

| Code | Meaning |
|------|---------|
| `401` | JWT missing or expired |
| `403` | Wallet has no manager role |
| `404` | Vault not found |
| `422` | Validation error in request body |
