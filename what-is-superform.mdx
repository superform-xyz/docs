# What is Superform?

Superform is a suite of persistent, non-upgradeable smart contracts that compress arbitrary on-chain actions into a single, Merkle-verified signature. Users interact through ERC-7579 Smart Accounts, and composition comes from plug-and-play Hooks that anyone can deploy.

## How It Works

1. **User signs one transaction** — the frontend builds a bundle of hooks (swap USDC → deposit into Morpho → bridge to Base) and presents it as a single signature request.
2. **SuperBundler packages the bundle** — splits hooks per chain, wraps each into an ERC-4337 UserOperation, and stores a Merkle tree of all authorized operations.
3. **Smart Account validates** — the on-chain SuperValidator checks the Merkle proof. If valid, the SuperExecutor iterates through hooks atomically.
4. **Accounting updates** — SuperLedger records cost basis and position changes for trustless performance fee calculation.

If any hook in a chain fails, the entire transaction reverts. Atomicity is guaranteed.

## Core Components

### Execution Layer

ERC-7579 executors (SuperExecutor, SuperDestinationExecutor) process hook bundles with transient storage for gas-efficient inter-hook communication. Same-chain bundles are atomic; cross-chain bundles use bridge adapters with destination validators.

### Validation Layer

Merkle-proof validators (SuperValidator, SuperDestinationValidator) enable single-signature authorization for batched multi-chain operations. One Merkle root encodes every permitted action — no per-transaction approval needed.

### Accounting Layer

SuperLedger and YieldSourceOracles provide trustless cost basis tracking and performance fee calculation across vault standards (ERC-4626, ERC-5115, ERC-7540, Pendle).

### Infrastructure

- **Bridge adapters** for cross-chain messaging (LayerZero, Hyperlane, CCIP)
- **SuperNativePaymaster** for ERC-20 gas sponsorship (pay gas in USDC)
- **SuperBundler** for batched UserOperation processing

## Who Uses Superform

| User | What They Do |
|------|-------------|
| **Depositors** | Earn yield through optimized vaults with a single deposit |
| **Curators** | Manage vault strategies — select yield sources, set allocation priorities, automate rebalancing |
| **Hook developers** | Build new on-chain modules for any DeFi protocol |
| **Validators** | Stake $UP to secure price-per-share oracle updates |
| **Integrators** | Embed Superform vaults and execution in their own products |

## Open and Permissionless

Anyone can:
- Deploy Superform Core contracts
- Write and register new hooks
- Create a SuperVault
- Build on top of the protocol

The contracts are non-upgradeable and fully open-source. No admin keys, no proxy upgrades, no trusted setup.
